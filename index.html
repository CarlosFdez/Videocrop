A user interface application used to crop videos on keyframes without retranscoding to maintain quality, remuxing the video into an mp4. Its similar to one of Avidemux's use cases, but supports exporting multiple intervals simultaneously and comes with video playback features.

Because it only cuts on keyframes, the cuts are not exact. The exported file will usually be slightly larger than the specified range.

Directions

To open a file, drag a video file to the open window.

Afterwards, you can edit the text field manually or use the Adjust Range buttons to update the ranges to export (indicated by blue boxes), and click the export button to export. It can support almost any input format, but the export is only to mp4.

How does it work?

A video is composed of a container consisting of several streams, where each stream is a video or audio track. What the application is replace the container for a new one and copy the desired segment of the stream. However, due to the format that most video streams are encoded, the segment is not exact, and will usually start before and end after the given range.

To understand why the cuts are not exact, you would need to understand interframe compression.

This application is a frontend for FFmpeg, and works under-the-hood by executing the following ffmpeg command for each export range:

ffmpeg -ss start -i input_file -avoid_negative_ts make_zero -t duration -map 0 -codec copy output_file
